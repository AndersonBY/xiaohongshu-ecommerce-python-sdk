## 商品基础介绍

### 一、商品业务基础说明

#### 1、业务简介

第三方开发者（ISV）和自研商家系统可以通过商品接口查询、创建、修改、删除商品信息。

#### 2、小红书商品ID

小红书商品ID是由数字和小写字母组成的编码，是商品的唯一识别码，可在店铺后台查看。  
【商家后台】-【商品】-【商品管理】-【商品列表】，商品标题下方的“商品id:"后的标号即是商品id。  
![](https://picasso-static.xiaohongshu.com/fe-platform/ea60ec68b76758cf73b66dc0ffde4d4119e258d6.png)

#### 3、SKU （Stock Keeping Unit）具有商品库存的最小单位

可简单理解为选定的一个可下单规格即是一个SKU。例如：下图水杯“黑色300ML—单杯+杯刷+3D贴纸“就是一个SKU，图中每一个颜色分类和套装明细的组合都是一个SKU。  
SKU编码可以自己设置，店铺后台【商品管理】-【商品列表】中选择编辑，在【销售信息】-【价格及库存】中的商品编码进行设置。

#### 4、SKU状态

小红书的商品在发布创建的时候会根据商家设置的规格属性组合出所有的产品sku，商家可以选择是否启用该sku，未启用的产品则不会上线，但后续可以通过编辑进行启用该sku进行上线。  
![](https://picasso-static.xiaohongshu.com/fe-platform/d79e0f038238007bfc3661508db9d590655e0044.png)  
**注意**：sku一经启用则无法关闭，后续如需下架则可以通过下架进行处理。

#### 5、商品价格

小红书商品价格精确到sku规格维度，商品各个SKU均需设置相应的价格  
可在店铺后台【商品管理】-【商品列表】中选择编辑，在【销售信息】-【价格及库存】中的售价进行设置价格。

#### 6、商品库存

商品库存需以SKU维度进行设置，指该SKU的可销售数量。  
平台商品库存是指平台商品SKU级别的库存，相同商品规格不同，如：1件装、多件装等，但只需输入一件装库存，系统会自动计算多件装库存，实际进行库存管理时需注意平台库存数与商家实际货品数的关系。例如共有10000个A商品库存，那么1件装的库存就是10000/1，而10件装的库存就是10000/10。  
可在店铺后台【商品管理】-【商品列表】中选择编辑，在【销售信息】-【价格及库存】中进行库存变动和库存设置。

#### 7、商品上下架状态

商家发布商品可以选择上架或者下架，上架商品买家才可选购；下架的商品会在【商品管理】-【商品列表】的商品状态列中显示。可以直接在操作栏中上架/下架商品所有SKU：  
![](https://picasso-static.xiaohongshu.com/fe-platform/d86b4ea94ce42353c953ef4ed337e46b65f28632.png) ![](https://picasso-static.xiaohongshu.com/fe-platform/7cd6c2fd19a5d9606b1c795677b29776fe5e5f66.png) ![](https://picasso-static.xiaohongshu.com/fe-platform/7cd6c2fd19a5d9606b1c795677b29776fe5e5f66.png)  
也可以在【商品管理】-【商品列表】中选择编辑，在【销售信息】-【价格及库存】中单独管理商品各个SKU的上下架状态，商品上架时，各SKU展示情况视各自的上下架状态而定。
